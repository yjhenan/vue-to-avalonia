## 4.1 MVVM模式深度解析

### 4.1.1 MVVM vs Vue的MVVM

**Vue的MVVM实现机制剖析**

Vue自称是一个MVVM框架，但实际上它是一种简化的实现。让我们回顾Vue的架构：

```vue
<template>
  <!-- View层：HTML模板 -->
  <div>
    <p>{{ message }}</p>
    <button @click="changeMessage">改变消息</button>
  </div>
</template>

<script setup>
// ViewModel层：Vue组件
import { ref } from 'vue'

const message = ref('Hello Vue')  // Model层数据

function changeMessage() {
  message.value = 'Message Changed'
}
</script>
```

**Vue的工作流程：**
```
用户操作 → View（模板）
    ↓
ViewModel（Vue组件）← 数据绑定 → Model（ref/reactive）
    ↓
自动更新View
```

**Vue的特点：**
- ViewModel和View在同一个.vue文件中
- 数据变化自动触发视图更新（响应式）
- 双向绑定通过`v-model`实现
- 开发者不需要关心View和Model如何同步

**Avalonia的MVVM严格分层**

Avalonia采用更严格的MVVM架构，三层分离更清晰：

```
┌─────────────────┐
│   View (.axaml) │  ← XAML文件，纯UI描述
└────────┬────────┘
         │ Binding
┌────────▼────────┐
│ ViewModel (C#)  │  ← C#类，业务逻辑和状态
└────────┬────────┘
         │
┌────────▼────────┐
│   Model (C#)    │  ← 数据模型、服务
└─────────────────┘
```

**实际示例：**

**View（MainWindow.axaml）：**
```xml
<Window xmlns="https://github.com/avaloniaui"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        xmlns:vm="clr-namespace:MyApp.ViewModels"
        x:Class="MyApp.Views.MainWindow"
        x:DataType="vm:MainViewModel">

  <StackPanel Margin="20">
    <!-- 绑定到ViewModel的Message属性 -->
    <TextBlock Text="{Binding Message}"
               FontSize="24"/>

    <!-- 绑定到ViewModel的ChangeMessageCommand -->
    <Button Content="改变消息"
            Command="{Binding ChangeMessageCommand}"
            Margin="0,20,0,0"/>
  </StackPanel>

</Window>
```

**ViewModel（MainViewModel.cs）：**
```csharp
using CommunityToolkit.Mvvm.ComponentModel;
using CommunityToolkit.Mvvm.Input;

namespace MyApp.ViewModels
{
    public partial class MainViewModel : ObservableObject
    {
        [ObservableProperty]
        private string message = "Hello Avalonia";

        [RelayCommand]
        private void ChangeMessage()
        {
            Message = "Message Changed";
        }
    }
}
```

**View Code-behind（MainWindow.axaml.cs）：**
```csharp
using Avalonia.Controls;
using MyApp.ViewModels;

namespace MyApp.Views
{
    public partial class MainWindow : Window
    {
        public MainWindow()
        {
            InitializeComponent();
            // 设置DataContext（连接View和ViewModel）
            DataContext = new MainViewModel();
        }
    }
}
```

**为什么Avalonia需要更严格的MVVM？**

1. **可测试性**：ViewModel可以脱离View进行单元测试
2. **设计时支持**：Visual Studio可以在设计时显示数据
3. **关注点分离**：UI设计师和开发者可以并行工作
4. **代码复用**：ViewModel可以在不同平台共享（如移动端）
5. **类型安全**：编译时检查绑定路径

### 4.1.2 Model, View, ViewModel职责划分

**对比Vue的data/computed/methods/watch**

| Vue概念 | MVVM层级 | Avalonia对应 | 说明 |
|--------|---------|------------|------|
| `data()` | Model | ViewModel的私有字段 | 数据存储 |
| `ref/reactive` | Model | 属性（带通知） | 响应式数据 |
| `computed` | ViewModel | C#属性（只读） | 计算属性 |
| `methods` | ViewModel | 方法/Command | 业务逻辑 |
| `watch` | ViewModel | `partial void OnXxxChanged` | 数据监听 |
| `props` | View→ViewModel | 构造函数参数 | 父子通信 |
| `emits` | ViewModel→View | 事件/回调 | 子父通信 |

**完整的ViewModel示例：**

```csharp
using CommunityToolkit.Mvvm.ComponentModel;
using CommunityToolkit.Mvvm.Input;
using System;
using System.Collections.ObjectModel;
using System.Linq;

public partial class TodoViewModel : ObservableObject
{
    // ===== Model层：响应式数据 =====
    public ObservableCollection<TodoItem> Todos { get; }

    // ===== Model层：属性（类似Vue的ref） =====
    [ObservableProperty]
    private string newTodoTitle = string.Empty;

    [ObservableProperty]
    private string filter = "all"; // all, active, completed

    // ===== ViewModel层：计算属性（类似Vue的computed） =====
    public int RemainingCount => Todos.Count(t => !t.IsCompleted);

    public int TotalCount => Todos.Count;

    private int completedCount;
    public int CompletedCount
    {
        get => completedCount;
        private set => SetProperty(ref completedCount, value);
    }

    // ===== 构造函数 =====
    public TodoViewModel()
    {
        Todos = new ObservableCollection<TodoItem>();
        Todos.CollectionChanged += (s, e) => UpdateCompletedCount();
    }

    // ===== Commands（类似Vue的methods） =====
    [RelayCommand]
    private void AddTodo()
    {
        if (string.IsNullOrWhiteSpace(NewTodoTitle))
            return;

        Todos.Add(new TodoItem
        {
            Id = Guid.NewGuid(),
            Title = NewTodoTitle,
            IsCompleted = false
        });

        NewTodoTitle = string.Empty;
    }

    [RelayCommand]
    private void RemoveTodo(TodoItem todo)
    {
        Todos.Remove(todo);
    }

    [RelayCommand]
    private void SetFilter(string newFilter)
    {
        Filter = newFilter;
    }

    private void UpdateCompletedCount()
    {
        CompletedCount = Todos.Count(t => t.IsCompleted);
    }
}

// Model类
public class TodoItem
{
    public Guid Id { get; set; }
    public string Title { get; set; } = string.Empty;
    public bool IsCompleted { get; set; }
}
```

**Vue对比实现：**

```vue
<script setup>
import { ref, reactive, computed, watch } from 'vue'

// Model层：响应式数据
const newTodoTitle = ref('')
const filter = ref('all')
const todos = reactive([])

// ViewModel层：计算属性
const remainingCount = computed(() =>
  todos.filter(t => !t.completed).length
)

const totalCount = computed(() => todos.length)

const completedCount = computed(() =>
  todos.filter(t => t.completed).length
)

// ViewModel层：方法
function addTodo() {
  if (!newTodoTitle.value.trim()) return

  todos.push({
    id: Date.now(),
    title: newTodoTitle.value,
    completed: false
  })

  newTodoTitle.value = ''
}

function removeTodo(todo) {
  const index = todos.indexOf(todo)
  todos.splice(index, 1)
}

function setFilter(newFilter) {
  filter.value = newFilter
}
</script>
```

**ViewModel的生命周期管理**

在Avalonia中，ViewModel的生命周期与View绑定：

```csharp
// View中管理ViewModel
public partial class MainWindow : Window
{
    private readonly MainViewModel viewModel;

    public MainWindow()
    {
        InitializeComponent();

        // 创建ViewModel
        viewModel = new MainViewModel();

        // 绑定到DataContext
        DataContext = viewModel;

        // 窗口关闭时清理资源
        this.Closed += (s, e) =>
        {
            viewModel.Dispose(); // 如果ViewModel实现了IDisposable
        };
    }
}
```

**实际项目中的最佳实践：**

1. **使用依赖注入（推荐）：**

```csharp
// App.axaml.cs
public class App : Application
{
    public override void Initialize()
    {
        // 配置依赖注入
        var services = new ServiceCollection();
        services.AddSingleton<IDataService, DataService>();
        services.AddTransient<MainViewModel>();

        var provider = services.BuildServiceProvider();
        Resources["ServiceProvider"] = provider;
    }
}

// MainWindow.axaml.cs
public partial class MainWindow : Window
{
    public MainWindow()
    {
        InitializeComponent();

        // 从DI容器获取ViewModel
        var provider = (ServiceProvider)Application.Current.Resources["ServiceProvider"];
        DataContext = provider.GetRequiredService<MainViewModel>();
    }
}
```
