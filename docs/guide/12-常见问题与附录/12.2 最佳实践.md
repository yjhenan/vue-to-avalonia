### 12.2.1 代码组织

**推荐的项目结构：**

```
MyApp/
├── Models/              # 数据模型
├── ViewModels/          # ViewModel层
│   ├── MainViewModel.cs
│   └── ViewModelBase.cs
├── Views/               # View层
│   ├── MainWindow.axaml

├── Controls/            # 自定义控件

├── Themes/              # 主题和样式

```

### 12.2.3 架构建议

**1. 严格遵循MVVM**

✅ 正确：

```csharp
// ViewModel
public class MainViewModel
{
    public ReactiveCommand<Unit, Unit> SubmitCommand { get; }

    public MainViewModel(IUserService userService)
    {
        SubmitCommand = ReactiveCommand.CreateFromTask(
            async () => await userService.SubmitAsync()
        );
    }
}
```

❌ 错误：

```csharp
// Code-behind中写业务逻辑
public partial class MainWindow : Window
{
    private void OnSubmit(object sender, RoutedEventArgs e)
    {
        var result = _service.Submit();  // 违反MVVM
    }
}
```

**2. 使用依赖注入**

```csharp
// 配置服务
services.AddSingleton<IUserService, UserService>();
services.AddTransient<MainViewModel>();

// 使用
public MainViewModel(IUserService userService)
{
    // 通过构造函数注入
}
```

**3. 异步编程最佳实践**

```csharp
// ✅ 好：使用async/await
public async Task LoadDataAsync()
{
    IsLoading = true;
    try
    {
        var data = await _service.GetDataAsync();
        Items = new ObservableCollection<Data>(data);
    }
    finally
    {
        IsLoading = false;
    }
}

// ❌ 坏：阻塞线程
public void LoadData()
{
    var data = _service.GetDataAsync().Result;  // 死锁风险！
}
```

### 12.2.4 性能优化建议

1. **使用OneTime绑定**：只读数据不需要双向绑定
2. **虚拟化大数据**：超过100项启用虚拟化
3. **防抖和节流**：频繁操作使用`Throttle`
4. **延迟加载**：非关键UI延迟初始化
5. **异步操作**：长时间运行的任务使用`Task.Run`
