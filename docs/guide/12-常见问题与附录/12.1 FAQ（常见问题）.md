# 第十二章：常见问题与最佳实践

## 12.1 FAQ（常见问题）

### 12.1.1 Web开发者常见困惑

**Q1：为什么Avalonia没有v-if和v-show？**

A：Avalonia使用`IsVisible`属性控制可见性，对应Vue的`v-show`。元素仍保留在可视树中。

```xml
<!-- v-show效果 -->
<Border IsVisible="{Binding IsLoading}">
  <TextBlock Text="加载中..."/>
</Border>

<!-- v-if效果（需要用不同的方式） -->
<ContentControl Content="{Binding CurrentView}"/>
```

**Q2：如何实现v-for的key功能？**

A：Avalonia的`ItemsControl`自动管理项的标识，通常不需要手动指定key。如果需要优化性能，可以实现`IKeyIndexProvider`接口。

**Q3：为什么没有watch？**

A：使用ReactiveUI的`WhenAnyValue`或`Subscribe`：

```csharp
// Vue: watch(() => props.count, (newVal) => {})
// Avalonia:
this.WhenAnyValue(x => x.Count)
    .Subscribe(newVal => { });
```

**Q4：如何实现computed？**

A：使用C#属性 + 手动通知：

```csharp
// Vue: computed(() => firstName + ' ' + lastName)
// Avalonia:
public string FullName => $"{FirstName} {LastName}";

// FirstName和LastName的setter中触发
this.RaisePropertyChanged(nameof(FullName));
```

**Q5：为什么绑定不生效？**

常见原因：
1. 没有实现`INotifyPropertyChanged`
2. Binding Path写错
3. DataContext未设置
4. 使用了`x:DataType`但类型不匹配

**调试方法：**

```csharp
// 添加调试输出
this.WhenAnyValue(x => x.Property)
    .Subscribe(value => Console.WriteLine($"Property changed: {value}"));
```

### 12.1.2 性能问题

**Q6：列表滚动卡顿？**

A：启用虚拟化：

```xml
<ListBox VirtualizationMode="Simple">
  <!-- 大数据量 -->
</ListBox>
```

**Q7：应用启动慢？**

A：
1. 使用`dotnet publish -c Release`发布
2. 启用Trimming：`<PublishTrimmed>true</PublishTrimmed>`
3. 使用AOT：`<PublishAot>true</PublishAot>`

**Q8：内存占用高？**

A：
1. 检查事件订阅是否取消
2. 避免静态集合无限增长
3. 使用`using`语句释放资源

### 12.1.3 调试技巧

**Q9：如何查看绑定错误？**

A：在Output窗口查看绑定错误信息，或使用Snoop工具。

**Q10：ViewModel如何单元测试？**

A：

```csharp
[Fact]
public void TestViewModel()
{
    var vm = new MainViewModel();
    vm.AddTodoCommand.Execute().Subscribe();

    Assert.Equal(1, vm.Todos.Count);
}
```
