## 5.2 Command模式

Command模式是Avalonia MVVM架构中处理事件的核心方式。相比于Vue的methods，Command提供了更好的可测试性、可复用性和可组合性。

### 5.2.1 ICommand接口详解

**ICommand接口定义：**

```csharp
public interface ICommand
{
    // 是否可以执行（控制按钮的IsEnabled状态）
    bool CanExecute(object? parameter);

    // 执行命令
    void Execute(object? parameter);

    // CanExecute状态改变事件
    event EventHandler? CanExecuteChanged;
}
```

**基础实现（不使用框架）：**

```csharp
using System;
using System.Windows.Input;

public class SimpleCommand : ICommand
{
    private readonly Action execute;
    private readonly Func<bool>? canExecute;

    public event EventHandler? CanExecuteChanged;

    public SimpleCommand(Action execute, Func<bool>? canExecute = null)
    {
        this.execute = execute;
        this.canExecute = canExecute;
    }

    public bool CanExecute(object? parameter)
    {
        return canExecute?.Invoke() ?? true;
    }

    public void Execute(object? parameter)
    {
        execute();
    }

    // 手动触发CanExecuteChanged
    public void RaiseCanExecuteChanged()
    {
        CanExecuteChanged?.Invoke(this, EventArgs.Empty);
    }
}

// 使用
public class MainViewModel : INotifyPropertyChanged
{
    public ICommand ClickCommand { get; }

    private int count = 0;
    public int Count
    {
        get => count;
        set
        {
            count = value;
            OnPropertyChanged(nameof(Count));
            // 手动通知Command状态改变
            (ClickCommand as SimpleCommand)?.RaiseCanExecuteChanged();
        }
    }

    public MainViewModel()
    {
        ClickCommand = new SimpleCommand(
            execute: () => Count++,
            canExecute: () => Count < 10  // 最多点击10次
        );
    }

    public event PropertyChangedEventHandler? PropertyChanged;
    protected void OnPropertyChanged(string name) =>
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(name));
}
```

```xml
<Button Command="{Binding ClickCommand}"
        Content="{Binding Count, StringFormat='点击次数：{0}'}"/>
<!-- 当Count达到10时，按钮自动禁用 -->
```

### 5.2.2 ReactiveCommand（推荐使用）

ReactiveUI提供了更强大的`ReactiveCommand`，自动处理CanExecute。

**基础用法：**

```csharp
using ReactiveUI;
using System.Reactive;
using System.Reactive.Linq;

public class MainViewModel : ReactiveObject
{
    // 无参数Command
    public ReactiveCommand<Unit, Unit> SimpleCommand { get; }

    // 带参数Command
    public ReactiveCommand<string, Unit> ParameterCommand { get; }

    // 带返回值Command
    public ReactiveCommand<Unit, string> ResultCommand { get; }

    // 异步Command
    public ReactiveCommand<Unit, Unit> AsyncCommand { get; }

    // 条件CanExecute
    public ReactiveCommand<Unit, Unit> ConditionalCommand { get; }

    private int count = 0;
    public int Count
    {
        get => count;
        set => this.RaiseAndSetIfChanged(ref count, value);
    }

    private bool isEnabled = true;
    public bool IsEnabled
    {
        get => isEnabled;
        set => this.RaiseAndSetIfChanged(ref isEnabled, value);
    }

    public MainViewModel()
    {
        // 1. 简单Command
        SimpleCommand = ReactiveCommand.Create(() =>
        {
            Count++;
            Console.WriteLine($"点击了{Count}次");
        });

        // 2. 带参数Command
        ParameterCommand = ReactiveCommand.Create<string>(name =>
        {
            Console.WriteLine($"你好，{name}");
        });

        // 3. 带返回值Command
        ResultCommand = ReactiveCommand.Create(() =>
        {
            return DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss");
        });

        // 4. 异步Command
        AsyncCommand = ReactiveCommand.CreateFromTask(async () =>
        {
            IsLoading = true;
            await Task.Delay(2000);  // 模拟异步操作
            Console.WriteLine("异步操作完成");
            IsLoading = false;
        });

        // 5. 条件Command（CanExecute基于Observable）
        ConditionalCommand = ReactiveCommand.Create(
            execute: () => Console.WriteLine("条件满足，执行命令"),
            canExecute: this.WhenAnyValue(x => x.IsEnabled)  // 监听IsEnabled属性
        );

        // 订阅Command的执行结果
        ResultCommand
            .Subscribe(result =>
            {
                Console.WriteLine($"命令返回：{result}");
                LastResult = result;
            });

        // 处理Command异常
        AsyncCommand
            .ThrownExceptions
            .Subscribe(ex =>
            {
                Console.WriteLine($"命令执行出错：{ex.Message}");
                ErrorMessage = ex.Message;
            });
    }

    private string lastResult = string.Empty;
    public string LastResult
    {
        get => lastResult;
        set => this.RaiseAndSetIfChanged(ref lastResult, value);
    }

    private bool isLoading = false;
    public bool IsLoading
    {
        get => isLoading;
        set => this.RaiseAndSetIfChanged(ref isLoading, value);
    }

    private string errorMessage = string.Empty;
    public string ErrorMessage
    {
        get => errorMessage;
        set => this.RaiseAndSetIfChanged(ref errorMessage, value);
    }
}
```

**XAML绑定：**

```xml
<StackPanel Spacing="10" Margin="20">
  <!-- 简单Command -->
  <Button Content="简单命令"
          Command="{Binding SimpleCommand}"/>

  <!-- 带参数Command -->
  <Button Content="问候Alice"
          Command="{Binding ParameterCommand}"
          CommandParameter="Alice"/>

  <Button Content="问候Bob"
          Command="{Binding ParameterCommand}"
          CommandParameter="Bob"/>

  <!-- 异步Command -->
  <Button Content="异步操作"
          Command="{Binding AsyncCommand}"
          IsEnabled="{Binding !IsLoading}"/>

  <!-- 条件Command -->
  <Button Content="条件命令"
          Command="{Binding ConditionalCommand}"/>
  <CheckBox Content="启用命令"
            IsChecked="{Binding IsEnabled}"/>
</StackPanel>
```

### 5.2.3 Command vs Vue methods对比

**完整示例对比：**

**Vue实现：**

```vue
<template>
  <div class="controls">
    <button @click="increment" :disabled="count >= 10">
      增加（{{ count }}/10）
    </button>

    <button @click="decrement" :disabled="count <= 0">
      减少
    </button>

    <button @click="reset">
      重置
    </button>

    <button @click="addAmount(5)">
      +5
    </button>

    <button @click="asyncIncrement" :disabled="isLoading">
      异步增加
    </button>

    <p v-if="error" class="error">{{ error }}</p>
  </div>
</template>

<script setup>
import { ref } from 'vue'

const count = ref(0)
const isLoading = ref(false)
const error = ref('')

function increment() {
  count.value++
  error.value = ''
}

function decrement() {
  count.value--
  error.value = ''
}

function reset() {
  count.value = 0
  error.value = ''
}

function addAmount(amount) {
  count.value += amount
}

async function asyncIncrement() {
  isLoading.value = true
  error.value = ''

  try {
    await new Promise(resolve => setTimeout(resolve, 1000))
    count.value++
  } catch (e) {
    error.value = e.message
  } finally {
    isLoading.value = false
  }
}
</script>

<style scoped>
.error {
  color: red;
}

button:disabled {
  opacity: 0.5;
}
</style>
```

**Avalonia实现：**

```xml
<Window xmlns="https://github.com/avaloniaui"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        xmlns:vm="clr-namespace:MyApp.ViewModels"
        x:Class="MyApp.Views.MainWindow"
        x:DataType="vm:CounterViewModel"
        Title="Command示例"
        Width="400" Height="300">

  <Window.Styles>
    <Style Selector="TextBlock.error">
      <Setter Property="Foreground" Value="Red"/>
      <Setter Property="Margin" Value="0,10,0,0"/>
    </Style>
  </Window.Styles>

  <StackPanel Margin="20" Spacing="10">
    <!-- 增加 -->
    <Button Command="{Binding IncrementCommand}"
            Content="{Binding Count, StringFormat='增加（{0}/10）'}"/>

    <!-- 减少 -->
    <Button Command="{Binding DecrementCommand}"
            Content="减少"/>

    <!-- 重置 -->
    <Button Command="{Binding ResetCommand}"
            Content="重置"/>

    <!-- 带参数 -->
    <Button Command="{Binding AddAmountCommand}"
            CommandParameter="5"
            Content="+5"/>

    <!-- 异步 -->
    <Button Command="{Binding AsyncIncrementCommand}"
            Content="异步增加"/>

    <!-- 错误信息 -->
    <TextBlock Text="{Binding Error}"
               Classes="error"
               IsVisible="{Binding HasError}"/>
  </StackPanel>

</Window>
```

```csharp
using ReactiveUI;
using System.Reactive;
using System.Reactive.Linq;
using System.Threading.Tasks;

public class CounterViewModel : ReactiveObject
{
    // ===== 属性 =====
    private int count = 0;
    public int Count
    {
        get => count;
        set => this.RaiseAndSetIfChanged(ref count, value);
    }

    private string error = string.Empty;
    public string Error
    {
        get => error;
        set
        {
            this.RaiseAndSetIfChanged(ref error, value);
            this.RaisePropertyChanged(nameof(HasError));
        }
    }

    public bool HasError => !string.IsNullOrEmpty(Error);

    private bool isLoading = false;
    public bool IsLoading
    {
        get => isLoading;
        set => this.RaiseAndSetIfChanged(ref isLoading, value);
    }

    // ===== Commands =====
    public ReactiveCommand<Unit, Unit> IncrementCommand { get; }
    public ReactiveCommand<Unit, Unit> DecrementCommand { get; }
    public ReactiveCommand<Unit, Unit> ResetCommand { get; }
    public ReactiveCommand<int, Unit> AddAmountCommand { get; }
    public ReactiveCommand<Unit, Unit> AsyncIncrementCommand { get; }

    // ===== 构造函数 =====
    public CounterViewModel()
    {
        // 增加（CanExecute：Count < 10）
        IncrementCommand = ReactiveCommand.Create(
            () =>
            {
                Count++;
                Error = string.Empty;
            },
            this.WhenAnyValue(x => x.Count)
                .Select(c => c < 10)
        );

        // 减少（CanExecute：Count > 0）
        DecrementCommand = ReactiveCommand.Create(
            () =>
            {
                Count--;
                Error = string.Empty;
            },
            this.WhenAnyValue(x => x.Count)
                .Select(c => c > 0)
        );

        // 重置
        ResetCommand = ReactiveCommand.Create(() =>
        {
            Count = 0;
            Error = string.Empty;
        });

        // 带参数
        AddAmountCommand = ReactiveCommand.Create<int>(amount =>
        {
            Count += amount;
        });

        // 异步（CanExecute：!IsLoading）
        AsyncIncrementCommand = ReactiveCommand.CreateFromTask(
            async () =>
            {
                IsLoading = true;
                Error = string.Empty;

                try
                {
                    await Task.Delay(1000);
                    Count++;
                }
                catch (Exception ex)
                {
                    Error = ex.Message;
                }
                finally
                {
                    IsLoading = false;
                }
            },
            this.WhenAnyValue(x => x.IsLoading)
                .Select(loading => !loading)
        );

        // 全局异常处理
        AsyncIncrementCommand
            .ThrownExceptions
            .Subscribe(ex => Error = ex.Message);
    }
}
```

### 5.2.4 Command的高级用法

**1. 组合Command（ReactiveCommand.CombineLatest）：**

```csharp
// 多个条件同时满足才能执行
var canExecute = Observable.CombineLatest(
    this.WhenAnyValue(x => x.IsOnline),
    this.WhenAnyValue(x => x.HasPermission),
    this.WhenAnyValue(x => x.IsLoading).Select(b => !b),
    (online, permission, notLoading) =>
        online && permission && notLoading
);

SubmitCommand = ReactiveCommand.Create(
    () => Console.WriteLine("提交"),
    canExecute
);
```

**2. Command链式执行：**

```csharp
// 第一个Command完成后执行第二个
LoadDataCommand = ReactiveCommand.CreateFromTask(LoadData);

ProcessDataCommand = ReactiveCommand.CreateFromTask(() =>
{
    // 等待LoadDataCommand完成
    return LoadDataCommand.Execute();
});

// 或者使用SelectMany
LoadDataCommand
    .SelectMany(_ => ProcessData())
    .Subscribe(result => Console.WriteLine($"处理完成：{result}"));
```

**3. 取消Command执行：**

```csharp
using System.Reactive.Disposables;
using System.Reactive.Concurrency;

private CancellationTokenSource? cts;

public ReactiveCommand<Unit, Unit> LongRunningCommand { get; }
public ReactiveCommand<Unit, Unit> CancelCommand { get; }

public MainViewModel()
{
    LongRunningCommand = ReactiveCommand.CreateFromTask(async () =>
    {
        cts = new CancellationTokenSource();

        for (int i = 0; i < 100; i++)
        {
            cts.Token.ThrowIfCancellationRequested();
            await Task.Delay(100, cts.Token);
            Progress = i;
        }
    });

    CancelCommand = ReactiveCommand.Create(() =>
    {
        cts?.Cancel();
        Console.WriteLine("已取消");
    });
}
```

### 5.2.5 Command最佳实践

**✅ 推荐做法：**

```csharp
// 1. 使用ReactiveCommand，自动管理CanExecute
public ReactiveCommand<Unit, Unit> SaveCommand { get; }

// 2. 使用Observable控制CanExecute
SaveCommand = ReactiveCommand.Create(
    () => Save(),
    this.WhenAnyValue(x => x.HasChanges, x => x.IsValid)
        .Select(tuple =>
        {
            var (hasChanges, isValid) = tuple;
            return hasChanges && isValid;
        })
);

// 3. 处理异常
SaveCommand.ThrownExceptions
    .Subscribe(ex => ShowError(ex.Message));

// 4. 异步操作使用CreateFromTask
LoadCommand = ReactiveCommand.CreateFromTask(async () =>
{
    var data = await _service.GetDataAsync();
    Items = new ObservableCollection<DataItem>(data);
});
```

**❌ 避免的做法：**

```csharp
// ❌ 在Command中直接操作UI
BadCommand = ReactiveCommand.Create(() =>
{
    // 不要这样做！
    var button = this.Find<Button>("MyButton");
    button.Content = "Clicked";
});

// ❌ 不处理异常
var command = ReactiveCommand.CreateFromTask(async () =>
{
    await RiskyOperation();  // 可能抛出异常
});
// 没有订阅ThrownExceptions

// ❌ 阻塞式异步
BadAsyncCommand = ReactiveCommand.Create(() =>
{
    Task.Delay(1000).Wait();  // 会阻塞UI线程！
});
```

## 5.3 小结

本节深入学习了Command模式：

✅ **ICommand接口**：理解CanExecute和Execute机制
✅ **ReactiveCommand**：使用ReactiveUI简化Command创建
✅ **参数传递**：CommandParameter的使用
✅ **异步Command**：CreateFromTask处理异步操作
✅ **条件CanExecute**：基于Observable动态控制
✅ **高级用法**：组合、链式、取消Command

**关键对比：**

| Vue methods | Avalonia Command | 说明 |
|------------|-----------------|------|
| `methods: { click() {} }` | `ReactiveCommand.Create(() => {})` | 无参命令 |
| `@click="fn(arg)"` | `CommandParameter="arg"` | 带参命令 |
| `:disabled="condition"` | `canExecute` Observable | 条件启用 |
| `async fn()` | `CreateFromTask(async () => {})` | 异步命令 |
| 无直接对应 | `ThrownExceptions` | 异常处理 |
