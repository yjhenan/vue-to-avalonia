## 7.3 可复用组件开发最佳实践

### 7.3.1 组件设计原则

**1. 单一职责原则（SRP）**

✅ **好的设计：**

```xml
<!-- 单一职责：只负责显示用户头像 -->
<UserControl x:Class="MyApp.Controls.Avatar">
  <Ellipse Width="{Binding Size}"
           Height="{Binding Size}">
    <Ellipse.Fill>
      <ImageBrush Source="{Binding ImageSource}"/>
    </Ellipse.Fill>
  </Ellipse>
</UserControl>
```

❌ **不好的设计：**

```xml
<!-- 职责过多：头像 + 用户信息 + 操作按钮 -->
<UserControl x:Class="MyApp.Controls.UserCardWithEverything">
  <!-- 太多职责混在一起 -->
</UserControl>
```

**2. 可配置性**

```csharp
// 提供合理的默认值和配置选项
public class Button : UserControl
{
    // 颜色变体
    public ButtonVariant Variant { get; set; } = ButtonVariant.Primary;

    // 尺寸
    public ButtonSize Size { get; set; } = ButtonSize.Medium;

    // 图标（可选）
    public string? Icon { get; set; }

    // 是否加载中
    public bool IsLoading { get; set; }
}

public enum ButtonVariant
{
    Primary,
    Secondary,
    Danger,
    Success
}

public enum ButtonSize
{
    Small,
    Medium,
    Large
}
```

**3. 事件命名约定**

```csharp
// ✅ 好的命名
public event EventHandler<UserEventArgs> UserSelected;
public event EventHandler<ItemEventArgs> ItemDeleted;

// ❌ 不好的命名
public event EventHandler Selected;  // 不明确
public event EventHandler OnClick;   // 多余的"On"前缀
```

## 7.4 小结

本章深入学习了Avalonia的组件化开发：

✅ **UserControl**：创建可复用的自定义控件
✅ **依赖属性**：实现组件的Props输入
✅ **路由事件**：实现组件的Emits输出
✅ **ControlTemplate**：自定义控件外观模板
✅ **DataTemplate**：自定义数据展示模板

**组件化对比总结：**

| 概念 | Vue | Avalonia |
|-----|-----|----------|
| **组件定义** | `.vue`单文件 | `UserControl` |
| **Props** | `defineProps()` | `StyledProperty` |
| **Emits** | `defineEmits()` | `RoutedEvent` |
| **插槽** | `<slot>` | `ContentPresenter` |
| **作用域插槽** | Scoped slot | `DataTemplate` |
| **生命周期** | `onMounted`等 | 构造函数 + 事件 |
| **组合** | Composition API | MVVM + Service |
