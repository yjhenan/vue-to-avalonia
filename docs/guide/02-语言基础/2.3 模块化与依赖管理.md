## 2.3 模块化与依赖管理

### 2.3.1 NuGet vs npm

**npm常用命令：**

```bash
# 初始化项目
npm init -y

# 安装依赖
npm install axios
npm install -D typescript

# 安装所有依赖
npm install

# 更新依赖
npm update

# 查看已安装的包
npm list
```

**NuGet常用命令：**

```bash
# 安装包
dotnet add package Newtonsoft.Json

# 安装特定版本
dotnet add package Newtonsoft.Json --version 13.0.1

# 移除包
dotnet remove package Newtonsoft.Json

# 还原依赖
dotnet restore

# 更新包
dotnet add package Newtonsoft.Json

# 列出已安装的包
dotnet list package

# 搜索包
dotnet package search communitytoolkit.mvvm
```

**Visual Studio中管理NuGet包：**

1. 右键点击项目 → 「管理NuGet程序包」
2. 在「浏览」标签页搜索包
3. 点击「安装」

### 2.3.2 using vs import

**JavaScript/TypeScript导入：**

```typescript
// TypeScript
import { ref, reactive } from 'vue'
import axios from 'axios'
import type { User } from './types'
import * as utils from './utils'

// 动态导入
const module = await import('./heavy-module')
```

**C#的using：**

```csharp
// C#
using System;
using System.Collections.Generic;
using System.Linq;
using System.Threading.Tasks;
using Newtonsoft.Json;
using MyApp.Models;

// using别名
using Json = Newtonsoft.Json.JsonConvert;

// 全局using（C# 10+，所有文件共享）
// 在GlobalUsings.cs中：
global using System;
global using System.Collections.Generic;
global using System.Linq;

// 隐式using（在.csproj中启用）
// <ImplicitUsings>enable</ImplicitUsings>
```

**命名空间（Namespace）：**

```csharp
// 定义命名空间
namespace MyApp.Services
{
    public class UserService
    {
        // ...
    }
}

// 使用命名空间
using MyApp.Services;

var service = new UserService();
```

### 2.3.3 项目文件（.csproj）vs package.json

**package.json示例：**

```json
{
  "name": "my-vue-app",
  "version": "1.0.0",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "vue": "^3.3.0",
    "axios": "^1.4.0"
  },
  "devDependencies": {
    "typescript": "^5.0.0",
    "vite": "^4.3.0"
  }
}
```

**.csproj示例：**

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>WinExe</OutputType>
    <TargetFramework>net8.0</TargetFramework>
    <Nullable>enable</Nullable>
    <ImplicitUsings>enable</ImplicitUsings>
    <ApplicationIcon>Assets\avalonia-logo.ico</ApplicationIcon>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Avalonia" Version="11.0.0" />
    <PackageReference Include="Avalonia.Themes.Fluent" Version="11.0.0" />
    <PackageReference Include="CommunityToolkit.Mvvm" Version="8.4.0" />
  </ItemGroup>

  <ItemGroup>
    <!-- 项目引用（类似本地npm包） -->
    <ProjectReference Include="..\MyApp.Core\MyApp.Core.csproj" />
  </ItemGroup>
</Project>
```

**关键对照：**

| package.json字段 | .csproj元素 | 说明 |
|----------------|-----------|------|
| `name` | `ProjectName`（文件名） | 项目名称 |
| `version` | `Version` | 版本号 |
| `scripts` | `TargetFrameworks` + CLI命令 | 构建脚本 |
| `dependencies` | `PackageReference` | 运行时依赖 |
| `devDependencies` | 开发时通常不需要 | 开发工具 |
| - | `OutputType` | 输出类型（Exe/Library） |
| - | `TargetFramework` | 目标框架版本 |
| - | `Nullable` | 可空引用类型检查 |

**多目标框架（类似多环境构建）：**

```xml
<PropertyGroup>
  <!-- 支持多个.NET版本 -->
  <TargetFrameworks>net8.0;net7.0</TargetFrameworks>
</PropertyGroup>

<!-- 条件编译 -->
<ItemGroup Condition="'$(TargetFramework)' == 'net8.0'">
  <PackageReference Include="SomePackage" Version="2.0.0" />
</ItemGroup>

<ItemGroup Condition="'$(TargetFramework)' == 'net7.0'">
  <PackageReference Include="SomePackage" Version="1.0.0" />
</ItemGroup>
```
